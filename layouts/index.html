Development philosophy

Before beginning to implement any solutions, it is important to fully investigate the problem so that the root cause is solved instead of the symptom. After identifying the cause, all code should be written in an easily readable and testable style. Maintainability is always be a goal. If a solution is difficult to understand, then the solution is just a looming future problem.
